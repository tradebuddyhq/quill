Developer Tools

Quill ships with built-in developer tools that help you write cleaner, more reliable code. No extra setup or plugins required.

Formatter (quill fmt)

The Quill formatter automatically reformats your code to follow a consistent style. It fixes indentation, spacing, and layout while preserving your code's meaning.

Usage

-- Format a file (overwrites in place)
quill fmt myprogram.quill

What it does

  • Normalizes indentation to 2 spaces
  • Adds blank lines between function definitions and block statements
  • Ensures consistent spacing around operators
  • Converts comparison operators back to English (is greater than, is less than)
  • Formats boolean values as yes/no

Example

Before formatting:

name is "Sarah"
age   is 25
if age is greater than 18:
  say "adult"
to add a b:
  give back a+b

After quill fmt:

name is "Sarah"
age is 25

if age is greater than 18:
  say "adult"

to add a b:
  give back a + b

Static Analyzer (quill check)

The static analyzer catches common mistakes and potential bugs without running your code. It reports errors, warnings, and helpful hints.

Usage

-- Check a file for issues
quill check myprogram.quill

What it checks

Errors (will cause problems at runtime)
  • Wrong argument count — calling a function with too many or too few arguments
  • Return outside function — using give back outside of a to block
  • Empty import path — use "" with no path
Warnings (probably not what you intended)
  • Unused variables — defined but never read (prefix with _ to suppress)
  • Division by zero — dividing by a literal 0
  • Self-comparison — comparing a variable to itself (x is x)
  • Always-true/false conditions — if yes: or if no:
  • Infinite loops — while yes: with no exit
  • Negative array index — JavaScript doesn't support Python-style items[-1]
  • Tests without expectations — a test block with no expect
  • Expect outside test — expect used without a test wrapper
  • Duplicate properties/methods — same name defined twice in a class
Info (style suggestions)
  • Lowercase class names — describe dog: should be describe Dog:
  • Functions without return — might be missing a give back

Example output

Found 3 issue(s) in program.quill:

  line 2 [warning]: variable "temp" is defined but never used
    hint: prefix with _ to suppress this warning
  line 8 [error]: add() expects 2 arguments but got 3
  line 12 [warning]: division by zero

Browser Mode (--browser)

Quill can compile to browser-compatible JavaScript, giving you access to DOM manipulation, events, and local storage — all with readable English syntax.

Usage

-- Build for the browser
quill build app.quill --browser

Then include the output app.js in a <script> tag in your HTML.

Browser functions

FunctionDescription
select(selector)Select a DOM element
selectAll(selector)Select all matching elements
setText(el, text)Set element text
setHTML(el, html)Set element HTML
getValue(el)Get input value
setValue(el, val)Set input value
addClass(el, cls)Add CSS class
removeClass(el, cls)Remove CSS class
toggleClass(el, cls)Toggle CSS class
hide(el) / show(el)Hide/show element
onClick(el, fn)Handle click events
onInput(el, fn)Handle input changes
onKeyPress(el, fn)Handle key presses
onLoad(fn)Run when page loads
createElement(tag)Create new element
append(parent, child)Append child element
save(key, val)Save to localStorage
load(key)Load from localStorage
after(ms, fn)Run after delay
every(ms, fn)Run on interval

Example: Todo app

-- Select elements
input is select("#todo-input")
button is select("#add-btn")
list is select("#todo-list")

-- Add a todo when button is clicked
to addTodo:
  text is getValue(input)
  li is createElement("li")
  setText li text
  append list li
  setValue input ""

onClick button addTodo

Project Init (quill init)

Scaffolds a new Quill project in the current directory. It creates a quill.json manifest (with name, version, and an empty dependency list) and a starter main.quill file.

Usage

-- Create a new project in the current folder
quill init

What it creates

  • quill.json — project manifest with name, version, and dependencies
  • main.quill — minimal starter file you can run right away

Example

$ mkdir my-app && cd my-app
$ quill init
Created quill.json
Created main.quill

$ quill run main.quill
Hello, Quill!

Add Package (quill add)

Installs an npm package and records it in your project's quill.json dependencies. This lets you use third-party JavaScript libraries from Quill code.

Usage

-- Install a package
quill add lodash

Example

$ quill add chalk
Installing chalk...
Added chalk@5.3.0 to dependencies

Remove Package (quill remove)

Uninstalls an npm package and removes it from quill.json dependencies.

Usage

-- Remove a package
quill remove lodash

Example

$ quill remove chalk
Removed chalk from dependencies

Generate Docs (quill docs)

Parses a Quill source file and generates an HTML documentation page listing all functions, parameters, and comments.

Usage

-- Generate docs for a file
quill docs mylib.quill

This produces a mylib.html file in the current directory.

Example

$ quill docs utils.quill
Generated utils.html (3 functions documented)
