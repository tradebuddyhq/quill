Building Discord Bots with Quill

Quill makes it straightforward to build Discord bots with clean, readable syntax. Under the hood, Quill compiles to JavaScript and uses the discord.js library — so you get the full power of the Discord API with Quill's English-like code style.

Prerequisites

Before you start, make sure you have the following:

  1. Node.js (version 16.11 or later) — download from nodejs.org. You can check your version by running node --version in your terminal.
  2. Quill installed — see the Getting Started guide.
  3. A Discord bot token — you get this from the Discord Developer Portal. Follow these steps:
    1. Go to the Discord Developer Portal and click New Application.
    2. Give your application a name and click Create.
    3. In the left sidebar, click Bot.
    4. Click Reset Token (or Add Bot if it is a new application) and copy the token. Keep it secret.
    5. Under Privileged Gateway Intents, enable Message Content Intent if you want your bot to read message text.
    6. Go to OAuth2 → URL Generator, select the bot scope, choose the permissions your bot needs, and use the generated URL to invite the bot to your server.
Keep your token safe. Never commit your bot token to version control. Use a .env file and add it to your .gitignore.

Quick Start

The fastest way to get a Discord bot running is with the built-in scaffolding command:

quill discord my-bot
cd my-bot

This creates a new project with the following structure:

my-bot/
  bot.quill          -- Your bot code
  package.json       -- Node.js dependencies (discord.js)
  .env               -- Your bot token goes here
  .gitignore         -- Keeps .env out of version control

Step 1: Open .env and paste your bot token:

DISCORD_TOKEN=your-bot-token-here

Step 2: Install dependencies:

npm install

Step 3: Build and run your bot:

quill build bot.quill
node bot.js

Your bot should come online in your Discord server. Try typing !hello in a channel to see it respond.

Full Bot Example

Here is a complete bot with multiple commands. This example shows how to set up the client, handle events, and respond to messages:

use "discord.js" as Discord

-- Create a bot with common intents pre-configured
bot is createBot(process.env.DISCORD_TOKEN)

-- Log when the bot is ready
bot on "ready" with:
  say "Bot is online as {bot.user.tag}!"

-- Respond to messages
bot on "messageCreate" with msg:
  -- Ignore messages from other bots
  if msg.author.bot:
    give back nothing

  -- Simple greeting
  if msg.content is "!hello":
    msg.reply("Hello, {msg.author.username}!")

  -- Echo command
  if startsWith(msg.content, "!say "):
    text is msg.content.slice(5)
    msg.channel.send(text)

  -- Ping/pong
  if msg.content is "!ping":
    msg.reply("Pong!")

  -- Server info
  if msg.content is "!server":
    name is msg.guild.name
    count is msg.guild.memberCount
    msg.reply("Server: {name} with {count} members")

  -- User info
  if msg.content is "!whoami":
    user is msg.author.username
    id is msg.author.id
    msg.reply("You are {user} (ID: {id})")

  -- Roll a dice
  if msg.content is "!roll":
    result is randomInt(1, 6)
    msg.reply("You rolled a {result}!")
Tip: The createBot helper function creates a Discord client with common intents (Guilds, GuildMessages, MessageContent) already configured. If you need more control, you can create the client manually — see the manual setup section below.

Manual client setup

If you need to configure specific intents or other client options, you can create the Discord client manually instead of using createBot:

use "discord.js" as Discord

intents is [
  Discord.GatewayIntentBits.Guilds,
  Discord.GatewayIntentBits.GuildMessages,
  Discord.GatewayIntentBits.MessageContent,
  Discord.GatewayIntentBits.GuildMembers
]

options is {intents: intents}
bot is new Discord.Client(options)

-- Register your event handlers here...

bot.login(process.env.DISCORD_TOKEN)

Event Handling

Discord bots are event-driven. Your bot listens for things that happen (a message is sent, a user joins, the bot starts up) and runs code in response. In Quill, you use the on keyword to register event handlers.

ready — Bot is online

Fires once when the bot has connected to Discord and is ready to receive events.

bot on "ready" with:
  say "Logged in as {bot.user.tag}"
  say "Serving {bot.guilds.cache.size} servers"

messageCreate — A message is sent

Fires every time someone sends a message in a channel the bot can see. The msg parameter contains the message object.

bot on "messageCreate" with msg:
  -- Always check if the author is a bot to avoid loops
  if msg.author.bot:
    give back nothing

  say "{msg.author.username}: {msg.content}"

interactionCreate — A slash command or button is used

Fires when a user triggers a slash command, clicks a button, or uses a context menu. This is the modern way to handle bot commands on Discord.

bot on "interactionCreate" with interaction:
  if interaction.isChatInputCommand() is not yes:
    give back nothing

  if interaction.commandName is "ping":
    interaction.reply("Pong!")

  if interaction.commandName is "hello":
    name is interaction.user.username
    interaction.reply("Hello, {name}!")

guildMemberAdd — A user joins the server

Fires when a new member joins a server the bot is in. Useful for welcome messages.

bot on "guildMemberAdd" with member:
  channel is member.guild.systemChannel
  if channel:
    channel.send("Welcome to the server, {member.user.username}!")

guildMemberRemove — A user leaves the server

bot on "guildMemberRemove" with member:
  channel is member.guild.systemChannel
  if channel:
    channel.send("Goodbye, {member.user.username}.")

Slash Commands

Slash commands are the modern, recommended way to build bot commands on Discord. Users type /commandname and get autocomplete, descriptions, and parameter validation built in.

Registering slash commands

You register slash commands using the Discord REST API. Here is how to do it in Quill:

use "discord.js" as Discord
from "discord.js" use REST, Routes, SlashCommandBuilder

-- Define your commands
commands are [
  new SlashCommandBuilder()
    .setName("ping")
    .setDescription("Check if the bot is alive"),

  new SlashCommandBuilder()
    .setName("hello")
    .setDescription("Get a greeting from the bot"),

  new SlashCommandBuilder()
    .setName("roll")
    .setDescription("Roll a dice")
    .addIntegerOption(with option:
      option.setName("sides")
        .setDescription("Number of sides on the dice")
        .setRequired(no)
    )
]

-- Convert to JSON for the API
commandData are map_list(commands, with cmd: cmd.toJSON())

-- Register commands with Discord
rest is new REST().setToken(process.env.DISCORD_TOKEN)

try:
  say "Registering slash commands..."
  await rest.put(
    Routes.applicationCommands(process.env.CLIENT_ID),
    {body: commandData}
  )
  say "Slash commands registered!"
if it fails err:
  say "Error registering commands: {err}"
Where do I find my Client ID? Go to the Discord Developer Portal, select your application, and copy the Application ID from the General Information page. Add it to your .env file as CLIENT_ID.

Handling slash commands

Once registered, handle slash commands in the interactionCreate event:

bot on "interactionCreate" with interaction:
  if interaction.isChatInputCommand() is not yes:
    give back nothing

  match interaction.commandName:
    when "ping":
      interaction.reply("Pong!")

    when "hello":
      name is interaction.user.username
      interaction.reply("Hello, {name}!")

    when "roll":
      sides is interaction.options.getInteger("sides")
      if sides is nothing:
        sides is 6
      result is randomInt(1, sides)
      interaction.reply("You rolled a {result} (d{sides})")

Slash commands with options

Slash commands can accept typed parameters. Discord validates the input before your bot even sees it.

-- A command with a required string option
new SlashCommandBuilder()
  .setName("echo")
  .setDescription("Repeat a message")
  .addStringOption(with option:
    option.setName("message")
      .setDescription("The message to echo")
      .setRequired(yes)
  )

-- Handling it
if interaction.commandName is "echo":
  text is interaction.options.getString("message")
  interaction.reply(text)

Deployment

Once your bot is working locally, you will want to run it on a server so it stays online. Here are some tips for deploying your Quill Discord bot.

Build for production

Compile your Quill code to JavaScript:

quill build bot.quill

This produces a bot.js file that runs on any system with Node.js installed.

Use environment variables

Never hardcode your bot token. Use a .env file locally and set environment variables on your production server:

-- In your bot code
token is env.require("DISCORD_TOKEN")
bot is createBot(token)

Keep the bot running

Use a process manager like pm2 to keep your bot alive and restart it if it crashes:

# Install pm2 globally
npm install -g pm2

# Start your bot
pm2 start bot.js --name my-bot

# View logs
pm2 logs my-bot

# Restart after updates
quill build bot.quill
pm2 restart my-bot

Hosting options

Discord bots need a persistent process (not serverless). Good options include:

  • A VPS (DigitalOcean, Linode, Vultr) — full control, starts around $5/month
  • Railway — simple deployment from Git, free tier available
  • Fly.io — deploy with a Dockerfile, generous free tier
  • A Raspberry Pi — run it at home for free

Using Docker

You can use Quill's built-in deployment command to generate a Dockerfile:

quill deploy

Or write your own minimal Dockerfile:

FROM node:20-slim
WORKDIR /app
COPY package.json .
RUN npm install --production
COPY bot.js .
CMD ["node", "bot.js"]

Build and run:

docker build -t my-bot .
docker run -d --env-file .env my-bot
