Language Reference

A complete guide to everything you can write in Quill.

Comments

Comments start with -- and continue to the end of the line. They are ignored by the compiler.

-- This is a comment
name is "Alice"  -- This is also a comment

Variables

Create variables with is (singular) or are (plural). Quill uses both for readability -- they work exactly the same way.

name is "Alice"
age is 25
colors are ["red", "green", "blue"]
active is yes

Reassign variables by using is again:

count is 0
count is count + 1  -- count is now 1

Data types

TypeDescriptionExample
TextA string of characters"Hello, World!"
NumberInteger or decimal42, 3.14

NothingAbsence of a value (null)nothing

Nothing

nothing represents the absence of a value, similar to null in other languages. Functions that don't explicitly return a value give back nothing.

result is nothing

if result is nothing:
  say "No result yet"

Operators

Math operators

OperatorDescriptionExample
+Addition2 + 3 gives 5
-Subtraction10 - 4 gives 6
*Multiplication3 * 7 gives 21
/Division15 / 4 gives 3.75
%Modulo (remainder)10 % 3 gives 1

Unary minus

x is -5
y is -x  -- y is 5

Strings & interpolation

Strings are wrapped in double quotes. Use {expression} inside strings to interpolate values:

name is "World"
say "Hello, {name}!"        -- Hello, World!
say "2 + 2 = {2 + 2}"        -- 2 + 2 = 4
say "Length: {length(name)}"  -- Length: 5

Escape sequences: \" for a literal quote, \\ for a backslash, \n for a newline, \t for a tab.

Comparisons

Quill comparisons read like English:

QuillMeaningJavaScript equivalent
x is greater than yx > yx > y
x is less than yx < yx < y
x is equal to ystrict equalityx === y

x is not ynot equalx !== y
list contains xmembership testlist.includes(x)
age is 20

if age is greater than 18:
  say "Adult"

if age is equal to 20:
  say "Twenty!"

colors are ["red", "blue"]
if colors contains "red":
  say "Red is present"

Logic operators

OperatorDescription
andBoth conditions must be true
orAt least one condition must be true

age is 25
name is "Alice"

if age is greater than 18 and name is "Alice":
  say "Hello, adult Alice!"

if not age is less than 18:
  say "Not a minor"

Control flow

If / otherwise

Conditional execution. The body must be indented (2 spaces recommended).

temp is 72

if temp is greater than 80:
  say "It's hot!"
otherwise if temp is greater than 60:
  say "It's nice"
otherwise:
  say "It's cold"

Try / Catch

Handle errors gracefully with try and if it fails. The error variable after if it fails captures the error message.

try:
  result is riskyOperation()
if it fails error:
  say "Something went wrong: {error}"

You can use try/catch around any block of code that might fail:

try:
  data is readFile("config.json")
  say "Config loaded"
if it fails error:
  say "Could not load config, using defaults"
  data is defaultConfig()

Loops

For each

Iterate over items in a list:

fruits are ["apple", "banana", "cherry"]

for each fruit in fruits:
  say "I like {fruit}"

Use range() to loop over numbers:

for each i in range(1, 6):
  say i   -- prints 1, 2, 3, 4, 5

While

Repeat as long as a condition is true:

count is 0

while count is less than 5:
  say count
  count is count + 1

Break & Continue

Use break to exit a loop early, and continue to skip to the next iteration:

for each i in range(1, 10):
  if i is 5:
    break        -- stop the loop entirely
  say i          -- prints 1, 2, 3, 4
for each i in range(1, 6):
  if i is 3:
    continue     -- skip this iteration
  say i          -- prints 1, 2, 4, 5

Functions

Define functions with to. Parameters are listed by name (no parentheses or commas needed in the definition).

to greet name:
  say "Hello, {name}!"

greet("Alice")   -- Hello, Alice!

Return values

Use give back to return a value from a function:

to add a b:
  give back a + b

result is add(3, 4)
say result   -- 7

Multiple parameters

to fullName first last:
  give back "{first} {last}"

say fullName("John", "Doe")   -- John Doe

Lambda / Arrow functions

Create inline anonymous functions with with. These are useful for callbacks and short expressions:

double is with x: x * 2
say double(5)   -- 10

Lambdas can take multiple parameters:

add is with a, b: a + b
say add(3, 4)   -- 7

They work well with list operations:

numbers are [1, 2, 3, 4, 5]
doubled is map(numbers, with x: x * 2)
say doubled   -- [2, 4, 6, 8, 10]

Type annotations

Quill supports optional type annotations on function parameters and return types. Use as to annotate parameter types and -> to specify the return type:

to add a as number, b as number -> number:
  give back a + b

Type annotations are optional and do not change runtime behavior. They serve as documentation and can be used by tooling:

to greet name as text -> text:
  give back "Hello, {name}!"

to isAdult age as number -> boolean:
  give back age is greater than 17

Classes

Define classes with describe. Classes can have properties and methods:

describe Animal:
  name is ""
  sound is ""

  to speak:
    say "{name} says {sound}"

Inheritance

Use extends to create a class that inherits from another:

describe Dog extends Animal:
  sound is "Woof"

  to fetch item:
    say "{name} fetches the {item}!"

The child class inherits all properties and methods from the parent and can override them or add new ones.

Lists

Lists are ordered collections of values:

numbers are [1, 2, 3, 4, 5]
mixed are ["hello", 42, yes]
empty are []

Access by index

colors are ["red", "green", "blue"]
say colors[0]   -- red
say colors[2]   -- blue

Dot access

Use dot notation to access properties on objects:

say "hello".length   -- 5

Object literals

Create objects with curly braces and key-value pairs:

person is {name: "Alice", age: 25, active: yes}

say person.name    -- Alice
say person.age     -- 25

Objects can be nested:

config is {
  host: "localhost",
  port: 3000,
  db: {
    name: "myapp",
    pool: 5
  }
}

say config.db.name   -- myapp

Spread operator

Use ... to spread a list into another list or into function arguments:

first are [1, 2, 3]
second are [4, 5, 6]
combined are [...first, ...second]
say combined   -- [1, 2, 3, 4, 5, 6]

The spread operator also works to merge objects:

defaults is {color: "blue", size: 10}
overrides is {size: 20, bold: yes}
merged is {...defaults, ...overrides}
say merged   -- {color: "blue", size: 20, bold: yes}

Output

Use say to print values to the console:

say "Hello!"
say 42
say 2 + 2

Imports

Import other Quill files with use:

use "helpers.quill"

Selective imports

Use from to import only specific names from a file:

from "math.quill" use add, subtract
from "utils.quill" use formatDate, parseJSON

This brings only the listed definitions into scope, keeping your namespace clean.

Imports are resolved at compile time and bring definitions from the imported file into scope.
